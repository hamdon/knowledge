在别人的服务器搭建网站系统的时候，最痛苦的事，莫过于他的环境配置跑不起你的应用。这时候怎么办呢？是帮他们升级环境呢，还是叫他们购买一台新的服务器呢？如果升级环境，之前服务器的应用也要做对应的处理，这是很痛苦的事，如果叫商家再购买一台，有很多是不愿意的。当出现这种情况的时候，可以使用docker来解决。

### docker在这里的主要作用
1. 在不影响原来服务器环境的前提下，快速搭建一个全新的符合我们自己需要的环境
2. 在docker里面跑网站，可以起到一层保护层，就算给入侵了，也只是在docker容器里面执行，对原服务器不受影响。

#### 创建启动文件sh脚本
```
#vim /docker/command/restart_service.sh
```
内容
```
#!/bin/sh
#

docker stop $(docker ps -a -q)

docker rm $(docker ps -a -q)

docker run --name my-redis -v /docker/redis/data:/data -v /docker/redis/config/redis.conf:/usr/local/etc/redis/redis.conf:ro -p 6379:6379 -d redis redis-server /usr/local/etc/redis/redis.conf --appendonly yes

docker run --name php7-fpm --restart=always --privileged=true -v /var/run/docker.sock:/var/run/docker.sock -v $(which docker):/bin/docker -v /docker/www/web:/www -v /docker/php/php.ini:/usr/local/etc/php/php.ini:ro -v /docker/php/www.conf:/usr/local/etc/php-fpm.d/www.conf:ro -v /docker/php/logs/php.log:/var/log/php.log:rw -v /docker/php/php-fpm.conf:/usr/local/etc/php-fpm.conf:ro --link my-redis:my-redis  --link mysql8:mysql -d crunchgeek/php-fpm:7.2 php-fpm

docker run --name runoob-php-nginx -p 8001:80 -d --restart=always -v /docker/www/web:/usr/share/nginx/html:ro -v /docker/nginx/vhost:/etc/nginx/conf.d:ro -v /docker/nginx/nginx.conf:/etc/nginx/nginx.conf:ro -v /docker/nginx/log/:/etc/nginx/logs:rw --link php7-fpm:php nginx

docker run -p 33060:3306 -e MYSQL_ROOT_PASSWORD=C111111$ -v /docker/mysql/data:/var/lib/mysql:rw -v /docker/mysql/log:/var/log/mysql:rw -v /docker/mysql/config/my.cnf:/etc/mysql/my.cnf:rw -v /etc/localtime:/etc/localtime:ro --name mysql8 --restart=always -d mysql

```

        #listen [::]:80 ipv6only=on;

        root /usr/share/nginx/html/public;
        index index.php index.html index.htm;

        # Make site accessible from http://localhost/
        server_name localhost;

        access_log /etc/nginx/logs/access.log;
        error_log /etc/nginx/logs/error.log;

        location ~ ^(.*)\/\.svn\/
        {
          return 404;
        }


        location / {
                try_files $uri $uri/ /index.php?$query_string;
                index index.php index.html index.htm;
        }

        location ~ \.php$ {
                  #  proxy_set_header Host $host
                    fastcgi_pass   php:9000;
                 #  fastcgi_pass   unix:/dev/shm/php-cgi.sock;
                    fastcgi_index  index.php;
                    fastcgi_param  SCRIPT_FILENAME  /www/public/$fastcgi_script_name;
                    include        fastcgi_params;
        }
        location ~ .*\.(gif|jpg|jpeg|png|bmp|swf|woff2|woff|ttf)$
        {
            expires      30d;
         }
        location ~ .*\.(js|css)?$
        {
            expires      12h;
        }

}
